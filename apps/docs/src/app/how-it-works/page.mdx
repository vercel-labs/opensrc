# How It Works

## Overview

opensrc resolves packages from their registry, clones the source repository at the correct version tag, and caches it globally. Use `opensrc path` inside other commands to search, read, or explore the source.

```
rg "parse" $(opensrc path zod)
в†’ resolves npm registry в†’ finds GitHub repo + version tag
в†’ git clone --depth 1 в†’ cached at ~/.opensrc/repos/github.com/colinhacks/zod/v3.24.1/
в†’ prints path to stdout в†’ rg searches the source
```

## Resolution

When you run `opensrc path zod`, the following happens:

1. **Registry lookup** вЂ” queries the npm registry for the `zod` package metadata
2. **Repository detection** вЂ” extracts the `repository` field from `package.json` to find the source repo URL
3. **Version tagging** вЂ” maps the installed or latest version to a git tag (e.g. `v3.24.1`)
4. **Shallow clone** вЂ” runs `git clone --depth 1 --branch v3.24.1` for a fast, minimal checkout
5. **Cache storage** вЂ” stores the result in `~/.opensrc/repos/github.com/colinhacks/zod/v3.24.1/`
6. **Path output** вЂ” prints the absolute path to stdout (progress goes to stderr so subshells work)

On subsequent runs for the same package version, steps 1-4 are skipped and the cached path is returned instantly. Multiple versions of the same package can be cached at once because each checkout lives under its versioned directory.

## Version Detection

For npm packages, opensrc tries to detect the version you're actually using in your project. It checks (in order):

1. `node_modules/<package>/package.json`
2. `package-lock.json`
3. `pnpm-lock.yaml`
4. `yarn.lock`
5. Falls back to the `latest` tag from the registry

This means the cached source matches the version in your project, not just the latest release.

```markdown
\`\`\`
```
