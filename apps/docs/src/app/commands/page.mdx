# Commands

## path

Print the absolute path to a package's source code, fetching automatically on cache miss. Designed for composing with other tools:

```bash
rg "parse" $(opensrc path zod)
cat $(opensrc path zod)/src/types.ts
find $(opensrc path pypi:requests) -name "*.py"
ls $(opensrc path crates:serde)/src/
grep -r "Router" $(opensrc path vercel/next.js)/packages/next/src/
```

Progress output goes to stderr, the path to stdout, so `$(opensrc path ...)` works in subshells.

| Flag | Description |
|------|-------------|
| `--cwd <path>` | Working directory for lockfile version resolution |
| `--verbose` | Show progress during fetch |

## fetch

Fetch and cache source code without printing the path. Useful for warming the cache with multiple packages at once:

```bash
opensrc zod react next
opensrc pypi:requests pypi:flask
opensrc crates:serde crates:tokio
opensrc vercel/ai
```

This is the default command when no subcommand is given.

## list

List all cached packages and repositories:

```bash
opensrc list
opensrc list --json
```

## remove

Remove a specific package or repository from the cache:

```bash
opensrc remove zod
opensrc remove pypi:requests
opensrc remove vercel/ai
```

## clean

Clean the entire cache, or filter by type:

```bash
opensrc clean
opensrc clean --packages
opensrc clean --repos
opensrc clean --npm
opensrc clean --pypi
opensrc clean --crates
```

| Flag | Description |
|------|-------------|
| `--packages` | Only clean packages (not repos) |
| `--repos` | Only clean repositories |
| `--npm` | Only clean npm packages |
| `--pypi` | Only clean PyPI packages |
| `--crates` | Only clean crates.io packages |
